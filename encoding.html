Encoding Methodology

Introduction

This digital scholarly edition of a selection of Anne Brontë's poetry has been produced alongside my written thesis on Brontë’s work as part of my PhD project. Anne Brontë's intriguing and layered body of poetry occupies a liminal space within the Brontë family oeuvre, being traditionally overlooked in terms of critical analysis. This edition aims to redress this gap by presenting a manuscript-centred edition accompanied by editorial analysis.

The following document outlines the methodology used during the creation of this digital scholarly edition of a selection of Brontë's poems. Step by step details about the encoding methodology and process are provided here to document the process of representing and implementing editorial decisions in a digital medium.

This edition features ten poems with themes relating to my thesis topic of childhood. The poems are presented in two separate editions. One is a diplomatic edition in which full diplomatic transcriptions of the poems are presented alongside the corresponding manuscript images, keeping focus on the texts of the manuscript versions. The critical edition of the poems presents a text which maintains the editorial preference for following the manuscript, but with an interface which provdes users with the option to examine the variations made by Brontë in later published versions.

The edition was prepared using the Oxygen XML Editor. The critical and diplomatic editions were prepared as separate XML files that adhere to the Text Encoding Initiative Guidelines. This standards-based method ensure platform-independent interoperability of the files and provides flexibility to produce a functional interactive edition that is inviting and user-friendly.

Definitions of different elements from the official TEI Guidelines have been included where appropriate throughout the document. I have provided links to the relevant sections of the TEI Guidelines, where appropriate.

Developing a consistent approach to the encoding of the edition was complex at times. There were certain fixed technical requirements, however, at times more creativity was needed to arrive at decisions which were both TEI compliant and achieved the desired visual effect when the edition is rendered online via Edition Visualization Technology (EVT). All of these decisions and methods are outlined here in a step by step guide to the editorial principles and construction of the edition. EVT is "a light-weight, open source tool specifically designed to create digital editions from TEI XML-encoded texts, freeing the scholar from the burden of web programming and enabling the final user to browse, explore and study digital editions by means of a user-friendly interface" (Turco et al).

Encoding process

TEI Header

In order to be compliant with the TEI Guidelines it is necessary to begin all documents with the element <teiHeader>, which must contain certain crucial information about the document, its text/s and encoding. While there are a number of possible elements which can be contained within <teiHeader>, the element <fileDesc> is the only one that is mandatory for all documents. [TEI Guidelines 2.2].

File description

The <fileDesc> element "contains a full bibliographic description of an electronic file" [2.2]. Within this element it is also mandatory to include the following elements : <titleStmt>, <publicationStmt> and <sourceDesc> [2.1].

Title statement

The element <titleStmt> "groups information about the title of a work and those responsible for its content" [2.2.1]. This must include the title of the file and usually also includes information on the authors and/or editors of the file as well as any other parties who are responsible for the production of the document. 

<fileDesc>
    <titleStmt>
        <title>Anne Brontë’s Poetry of Childhood: A Digital Scholarly Edition</title>
    </titleStmt>
</fileDesc>
Publication statement

The purpose of <publicationStmt> is “to name the agency by which a resource is made available (for example, a publisher or distributor) and to supply any additional information about the way in which it is made available such as licensing conditions, identifying numbers, etc.” [2.2.4]. 

<publicationStmt>
    <p>Publication Information</p>
    <publisher>University of Galway</publisher>
    <pubPlace>Galway, Ireland</pubPlace>
    <date>2024</date>
    <availability>
        <p>This is an open access work licensed under a Creative Commons Attribution 4.0
            International license.</p>
    </availability>
</publicationStmt>
Source description

The <sourceDesc> element "describes the source(s) from which an electronic text was derived or generated, typically a bibliographic description in the case of a digitized text" [2.2.7].

In the case of this edition, this primarily means including information about the textual witnesses, which include printed and manuscript documents. The <sourceDesc> element spans the breadth of bibliographic information of all witness and contains a variety of elements:

Structured bibliographic citation

The first element included within the <sourceDesc> is <biblStruct> which "contains a structured bibliographic citation, in which only bibliographic sub-elements appear and in a specified order" [3.11.1].

One examples is as follows:

<biblStruct>
    <monogr>
        <author>Anne Brontë</author>
        <title>"Tis Strange to Think"</title>
        <imprint>
            <date>1846</date>
        </imprint>
    </monogr>
</biblStruct>
Witnesses

As this is a digital critical edition, there are a number of different witnesses being documented and encoded. These witnesses are the original Anne Brontë manuscripts and any printed versions of the poems which appeared within Brontë's lifetime.

The information for the witnesses which are used in each poem is included in the <listWit> element which "lists definitions for all the witnesses referred to by a critical apparatus, optionally grouped hierarchically" [12.1.4.2.] <listWit> is included in the <sourceDesc> section of the <teiHeader>.

Each witness within <listWit> is identified by a <wit> element.

For example:

<witness xml:id="MS-Q">
    <listWit>
        <bibl type="manuscript">
            <title>Anne Brontë Manuscript 1842-5</title>
            <msIdentifier>
                <settlement>New York</settlement>
                <repository>Morgan Library and Musuem </repository>
                <idno>MA 28</idno>
            </msIdentifier>
        </bibl>
    </listWit>
</witness>
Witness id elements

Each witness needs to be encoded with a unique @xml:id identifier so that it can be linked to variations within the text. For the printed texts the sigla used were a form of initials for the text e.g. for the 1846 Poems by Currer, Ellis and Acton Bell:

<witness xml:id="ACE">

For the manuscripts I decided to adopt the sigla used by Edward Chitham in his 1979 comprehensive print edition of The Poems of Anne Brontë (updated and reissued in 2021) to facilitate easy cross-referencing of the manuscripts by readers. For example:

<witness xml:id="MS-P">

This id was then used when encoding variations to indicate which readings can be found in each witness:

<rdg wit="#ACE">,</rdg>
Witness information

When providing information about the text within the <wit> element, I used the <bibl> element to group relevant bibliographic information about the witnesses.

For manuscripts, this includes the manuscript title, with information about the repository where the manuscript is housed and its catalogue number included within the <msIdentifier> element:

<bibl type="manuscript">
    <title>Anne Brontë Manuscript, 1838-41</title>
    <msIdentifier>
        <settlement>New York</settlement>
        <repository>Morgan Library and Museum</repository>
        <idno>MA 2696.5</idno>
    </msIdentifier>
</bibl>

The information included in the <bibl> element for published books provides basic bibliographic information on the text:

<bibl type="book">
    <title>Poems by Currer, Ellis, and Acton Bell</title>
    <author>Charlotte Brontë</author>
    <author>Emily Brontë</author>
    <author>Anne Brontë</author>
    <date>1846</date>
    <publisher>Aylott and Jones</publisher>
</bibl>
